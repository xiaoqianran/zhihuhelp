# 知乎登录页与桌面启动器修复记录

## 背景

启动应用后进入登录页时，页面只显示“打开调试面板”按钮，没有显示知乎登录页。

排查后确认登录页依赖 Electron `webview` 渲染知乎页面。如果直接在普通浏览器中打开前端页面，或者 Tabs 内容区没有正确撑开，都会导致登录页区域不可见或不可用。

## 登录页修复

本次修复包含以下内容：

1. 登录页改为在登录 tab 激活时动态创建 `webview#zhihu-login`，目标地址为 `https://www.zhihu.com/signin`。
2. 为 `webview` 设置桌面浏览器 User-Agent，避免知乎页面因过旧 UA 或非预期环境展示异常。
3. 为登录页容器、Ant Design Tabs 内容区和 `webview` 设置明确宽高，避免只显示调试按钮。
4. Tabs 子页面改用 `React.createElement` 作为正常 React 组件渲染，避免直接调用组件函数带来的 hook/ref 生命周期问题。
5. Electron 主窗口保持启用 `webviewTag`，并在 Linux 下关闭 renderer sandbox，以保证 preload 和 webview 正常工作。
6. 拦截 `webview` 内的 `window.open`，让知乎登录流程中的新窗口跳转继续在当前 webview 内完成。

## 构建验证

执行过以下命令：

```bash
npm run buildgui
npm run build
```

两条命令均构建成功。`npm run buildgui` 会把前端构建结果同步到 `dist/client`。

还通过虚拟显示环境做过短时 Electron 启动烟测：

```bash
timeout 20s xvfb-run -a ./node_modules/.bin/electron --no-sandbox --disable-gpu --disable-dev-shm-usage dist/index.js --zhihuhelp-debug
```

烟测期间没有应用级崩溃。输出中的 DBus/GPU 报错来自当前虚拟显示环境。

## 桌面启动器

为了支持在桌面上双击启动，新增了仓库根目录下的启动器文件：

```text
知乎助手.desktop
```

当前启动器内容使用本机项目绝对路径：

```ini
[Desktop Entry]
Type=Application
Name=知乎助手
Comment=启动知乎助手
Exec=/root/Downloads/pro/zhihuhelp/node_modules/.bin/electron --no-sandbox --disable-gpu --disable-dev-shm-usage /root/Downloads/pro/zhihuhelp/dist/index.js --zhihuhelp-debug
Path=/root/Downloads/pro/zhihuhelp
Icon=/root/Downloads/pro/zhihuhelp/src/public/icon/icon.png
Terminal=false
Categories=Utility;
StartupNotify=true
```

创建并放到桌面的过程：

```bash
cp /root/Downloads/pro/zhihuhelp/知乎助手.desktop /root/Desktop/知乎助手.desktop
chmod +x /root/Downloads/pro/zhihuhelp/知乎助手.desktop /root/Desktop/知乎助手.desktop
```

如果桌面环境提示“不受信任的启动器”或“允许启动”，选择允许或信任即可。
