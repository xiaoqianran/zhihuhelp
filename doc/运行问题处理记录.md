# 运行问题处理记录

本文档记录本轮排查中已经遇到、定位和处理过的问题，后续继续维护。

## 2026-07-03

### Linux 下 js-rpc 签名窗口卡住

- 现象：Linux 执行抓取时停在“等待js-rpc签名窗口初始化”，知乎请求无法继续。
- 原因：签名窗口依赖嵌套 `webview`，Linux/Electron 环境下页面没有稳定初始化。
- 处理：改为直接通过 `loadFile()` 加载签名页面，并在当前页面初始化签名函数，移除嵌套 `webview` 依赖。
- 结果：签名窗口可以初始化，后续请求能继续执行。

### DevTools 断开弹窗干扰

- 现象：进入签名窗口时反复出现 `DevTools was disconnected from the page`。
- 原因：自动打开 DevTools 会在签名页重载/切换时触发断开提示。
- 处理：关闭自动 DevTools 行为，并补充渲染进程异常日志。

### 抓取失败后仍生成电子书

- 现象：抓取失败后仍显示“开始生成电子书”，最后输出空内容或不完整内容。
- 原因：命令错误没有向上层明确传播。
- 处理：抓取命令失败后记录全局错误并阻止生成；空任务、接口空数据、HTTP 错误改为抛错。

### 403/429 风控时继续派发任务

- 现象：触发知乎风控或请求限制后，后续任务仍继续跑。
- 原因：任务调度器没有把 403/429 作为立即停止信号。
- 处理：HTTP 层标记 403/429，任务调度器检测后停止继续派发，并输出明确日志。

### 继续上次模式仍从头处理大量已完成回答

- 现象：暂停后再次运行，会从前面大量已完成回答开始检查，速度慢且日志误导。
- 原因：只在回答详情层做本地存在检查，跳过任务还会计入保护休眠。
- 处理：回答、文章、想法详情抓取前先查数据库；跳过任务不再触发保护休眠。

### 首页缺少抓取方式选择

- 现象：需要弹窗或额外方式选择“继续上次/从头抓取”。
- 处理：首页任务配置增加“抓取方式”，默认“继续上次”，并写入 `fetchConfig.mode`。

### 继续上次模式仍请求问题信息和回答列表

- 现象：数据库已有回答时，第一轮仍会请求知乎问题信息和回答 ID 列表。
- 原因：续传跳过只在回答详情层生效，问题层没有先检查本地完整度。
- 处理：问题任务在继续上次模式下先查数据库；本地回答数量与记录总数差距在容忍范围内时，直接跳过问题信息和回答列表抓取。
- 补充：用户所有回答任务也增加同类本地完整度判断。

### EPUB 内容页过大导致阅读器显示异常

- 现象：生成速度很快，EPUB 打开后像是没有内容或没有拼接完整。
- 原因：问题类型把数百条回答合并到单个 HTML，部分 EPUB 阅读器兼容性差。
- 处理：生成 EPUB 时按 50 条回答拆分问题内容页，并在生成前校验内容记录数。
- 验证：问题 `31971136` 从单个大 HTML 拆分为 9 个 EPUB 内容页。

### 日志文案误导

- 现象：“添加抓取任务”容易被理解为已经向知乎发送请求。
- 处理：通用任务日志改为“处理任务”；真正网络请求以“准备生成知乎请求签名”“准备抓取...”为准。

### 待继续关注

- 历史日志中已写入过完整 cookie，后续需要避免继续输出敏感配置。

### HTTP 内存缓存命中前仍等待签名

- 现象：同一进程内即使接口响应已进入内存缓存，后续请求仍会先等待 js-rpc 生成签名。
- 原因：缓存命中判断写在签名生成之后。
- 处理：先计算最终 URL 和缓存 key，命中缓存时直接返回，不再签名、不再发网络请求。

### 日志输出完整 cookie

- 现象：抓取/生成命令把完整 `config.json` 写入日志，包含 cookie。
- 原因：读取配置后直接输出原始文本。
- 处理：新增配置日志脱敏，后续日志中 cookie 显示为 `[已隐藏cookie]`。

### 更多列表任务续传仍会刷新列表

- 现象：用户文章、用户想法、专栏、话题在本地已有足够数据时，仍会请求信息和列表接口。
- 原因：这些任务只依赖详情层跳过，没有在列表层做本地完整度判断。
- 处理：为用户文章、用户想法、专栏、话题增加本地完整度判断，数量足够时直接跳过信息和列表抓取。
- 暂不处理：收藏夹列表总数语义不够稳定，避免误判漏抓，暂不做整任务跳过。

### 想法接口空数据仍写库

- 现象：想法详情接口返回空对象时仍继续写库。
- 处理：想法详情与回答、文章保持一致，空数据直接抛错并停止。

### 调试输出可能泄漏配置或制造噪音

- 现象：主进程和前端仍残留若干 `console.log`，其中主进程曾输出完整配置对象。
- 处理：移除主进程配置调试输出、前端任务提交/登录检查调试输出、登录页导航调试输出。

### 话题精华回答关系写库失败被吞掉

- 现象：保存话题与回答关系时捕获错误后只打印 `console.log`，任务仍继续成功。
- 处理：移除吞错逻辑，写库失败会按任务失败向上抛出。
